Methods

This project is a work in progress and the results should be read as preliminary.

This project analyses co-occurrence patterns of plant taxa across space, time, and biological resolution using fossil pollen records, modern vegetation records, co-located climate data, site coordinates, and plant functional traits. The central aim is to quantify how much of the observed co-occurrence signal is explained by shared abiotic responses, spatial or spatiotemporal structure, and residual species-species association signals.

The project is part of the BIODYNAMICS initiative.

Data Sources

VegVault

All primary vegetation, climate, coordinate, and trait inputs are sourced from VegVault, a harmonised SQLite database that combines fossil pollen archives, modern vegetation datasets, co-registered climate reconstructions, and plant functional traits. Using a single harmonised data source keeps site identifiers, sample ages, coordinates, taxonomy, and climate values consistent across the paleo, modern, and trait workflows. For each analysis, records are extracted for the spatial window, temporal range, data source, and predictor set defined by the active configuration in config.yml.

NoteTechnical details - data extraction

The function extract_data_from_vegvault() queries a local VegVault SQLite database and returns a unified tidy object containing community and abiotic records for the requested analysis. Coordinates are extracted with get_coords(), which records one coordinate pair per dataset rather than one coordinate pair per sample. Database presence and accessibility are checked at session startup with check_presence_of_vegvault().

Configuration-controlled extraction filters:

  • Spatial window: x_lim and y_lim
  • Temporal window: age_lim
  • Dataset types: sel_dataset_type
  • Abiotic variables: sel_abiotic_var_name
  • Target store root: target_store

Climate Variables

Climate predictors represent the shared abiotic response component of the joint species distribution models. Paleo analyses use CHELSA-TraCE21k bioclimatic variables co-registered with the pollen records in VegVault. The broad candidate set includes variables describing mean thermal energy, temperature seasonality and extremes, annual precipitation, and seasonal precipitation.

To prevent redundant predictors from inflating coefficient uncertainty, the pipeline screens candidate climate variables before model fitting. The current workflow removes invariant predictors, evaluates collinearity, and retains a non-redundant predictor subset for the active dataset.

NoteTechnical details - abiotic processing

Pipe segment: pipe_segment_abiotic_extract.R

Predictor screening and scaling are performed inside each target store, so each spatial unit, time-slice workflow, and data source can have its own model-ready predictor matrix.

Plant Functional Traits

Plant traits are used to build functional-type classifications that can be compared with genus- and family-level community matrices. The traits workflow extracts raw trait values from VegVault, flags problematic values, applies curated corrections, aggregates validated trait values to taxa, and clusters taxa into functional types. Functional types are generated as reference classifications and then consumed by the spatial-resolution pipelines.

Key helper functions include extract_traits_from_vegvault(), clean_raw_trait_data(), apply_trait_corrections(), aggregate_trait_values(), and cluster_functional_types().

Analysis Axes

The current project structure is broader than the original two-pipeline design. The analysis now spans paleo and modern data sources, three spatial scales, temporal slices through the late Quaternary, and multiple taxonomic or functional resolutions.

ImportantComplementary analysis axes

Temporal - How do co-occurrence patterns change through time?

pipeline_paleo_temporal.R fits the shared modelling core independently across paleo time slices. Each branch receives a target age from tarchetypes::tar_map(), filters the aligned sample set to that time slice, fits a model, calculates variance decomposition, and summarises temporal trajectories.

Configurations: project_paleo_temporal_europe, project_paleo_temporal_america, project_paleo_temporal_asia

Spatial - How do co-occurrence patterns vary across geographic scale?

pipeline_paleo_spatial_resolution.R and pipeline_modern_spatial_resolution.R run the shared modelling core independently for geographic units defined in Data/Input/spatial_grid.csv. Spatial units are stored separately with store_suffix, so each unit has its own target store and progress graph.

Configurations: project_paleo_spatial_continental, project_paleo_spatial_regional, project_paleo_spatial_local, project_modern_spatial_continental, project_modern_spatial_regional, project_modern_spatial_local

Biological resolution - Do conclusions depend on how taxa are grouped?

Spatial-resolution pipelines branch over community resolution. Paleo spatial analyses compare genus, family, and functional-type matrices. Modern spatial analyses support genus, family, and functional-type variants routed through the current branch labels.

Resolution branches: genus, family, and functional type.

The analytical core is shared across these axes. The main differences are how samples are selected, which community resolution is supplied to the model, and whether the spatial component is purely spatial or spatiotemporal.

Pipeline Stages

1. VegVault Data Extraction

Vegetation records, climate data, and coordinates are queried from a local VegVault SQLite database for the active configuration. The extraction produces parallel data objects that remain separate through preprocessing and are only merged at sample alignment and model assembly.

NoteTechnical details

Pipe segment: pipe_segment_vegvault_extract.R

Target Function Description
data_vegvault_extracted extract_data_from_vegvault() Raw community and abiotic records from VegVault
data_coords get_coords() One coordinate pair per dataset_name
check_n_cores check_min_n_cores() Guard for minimum available compute resources

The extraction segment is reused by paleo, modern, and development pipelines. Target store isolation is controlled by the active configuration and, for spatial workflows, by store_suffix.

2. Community Data Extraction

Community data are extracted from the VegVault result object and converted into the long-format structure expected by downstream preprocessing. This stage keeps the raw taxon names and sample metadata available for classification, interpolation, filtering, and audit checks.

NoteTechnical details

Pipe segment: pipe_segment_community_extract.R

Step Target or helper Role

Long conversion make_community_data_long() Convert community data to tidy long format
Age attachment add_age_to_samples() Attach sample ages before interpolation or filtering
Pollen sums get_pollen_sum() Calculate pollen sums used in proportional filtering

3. Taxonomic Classification

Taxonomic classification resolves source taxon names to a consistent project taxonomy. The workflow combines automated classification, project-specific auxiliary tables, and missing-taxon reporting so unresolved or manually corrected cases remain explicit.

NoteTechnical details

check_and_report_missing_taxa() Produces explicit missing-classification reports

This stage supports the later genus, family, and functional-type branches.

4. Paleo Community Preparation

Paleo community preparation transforms fossil pollen counts into a filtered, classified, and temporally standardised community table. The workflow converts counts to proportions, propagates sample ages, extracts age uncertainty, interpolates irregular records to the configured temporal grid, removes non-plant taxa, resolves taxonomic resolution, and applies data-density filters.

Age uncertainty and interpolation are central to the paleo workflow because fossil samples are irregularly spaced and have uncertainty from age-depth modelling.

5. Modern Community Preparation

Modern community preparation shares the same broad goal as the paleo workflow but has different quality-control problems. Modern vegetation data are checked for impossible values, exact duplicates, colocated records, and source-specific aggregation cases before they are filtered and supplied to the shared modelling stack.

Cross-database colocations are tracked rather than silently merged unless the aggregation rules explicitly allow it. This keeps modern data processing auditable.

6. Community Resolution Routing

After shared community preparation, spatial-resolution pipelines branch over biological resolution. The same input data can be routed to genus-level, family-level, or functional-type community matrices before alignment and model fitting.

Resolution Main operation

Functional-type classifications are generated before the resolution branches need them. Spatial-resolution pipelines read those classifications with helpers such as get_functional_type_classification_path_from_store() and read_functional_type_classification().

7. Data Alignment

Data alignment defines which samples enter the model. The pipeline retains only sample identifiers that can be matched across community data, abiotic predictors, and coordinates.

NoteTechnical details

Spatial segment: pipe_segment_sample_alignment.R

Temporal segment: pipe_segment_sample_filter_age.R

Path Sample-selection logic
Spatial align_sample_ids() computes the intersection across community, abiotic, and coordinate records for the whole spatial unit
Temporal The time-slice branch supplies an age value and filters aligned samples to that target slice
Validation check_data_sample_ids_n() checks whether enough samples remain

All model matrices use a consistent sample-key convention derived from dataset_name and age. This allows community, abiotic, coordinate, and spatial predictor matrices to be checked for identical row order before fitting.

8. Model Input Preparation

The aligned data are converted into matrix form before fitting. Community data are made wide and binarised, abiotic predictors are prepared and scaled, coordinates are projected, and spatial predictors are calculated.

NoteTechnical details

For spatial analyses, MEMs are computed from geographic distances among sites. For temporal analyses, MEMs can be computed from sample-level spatiotemporal coordinates so that broad temporal and spatial autocorrelation are handled together.

9. Joint Species Distribution Model

The model is a Joint Species Distribution Model fitted with {sjSDM}. This framework fits all taxa jointly and estimates residual covariance among taxa after accounting for shared climate and spatial structure. The residual covariance is the association signal of interest, but it is interpreted cautiously because unmeasured environment, dispersal, history, and true interactions can all contribute to it.

NoteTechnical details

Pipe segment: pipe_segment_model_fit.R

Target or function Description
make_env_formula() Builds the environmental model formula
fit_jsdm_model() Fits the {sjSDM} model
compute_jsdm_se() Computes post-hoc standard errors
evaluate_jsdm() Summarises model fit and species-level performance
check_convergence_jsdm() Checks model convergence diagnostics
check_cuda_gpu_runtime() Checks GPU runtime availability

Model components:

  • Environment: non-redundant climate predictors and, where configured, interactions with age.
  • Space: MEM predictors used as a spatial or spatiotemporal structure component.
  • Association: residual low-rank covariance structure estimated by {sjSDM}.
  • Response: binary presence/absence community matrix.

The project previously used HMSC-style MCMC modelling. The workflow was migrated to {sjSDM} because gradient-descent fitting with optional GPU support is more practical for the large community matrices used here.

10. Variance Decomposition

After fitting, the model is decomposed into abiotic, spatial, and association components. This is the main comparative quantity used across spatial scale, time, data source, and biological resolution.

NoteTechnical details

Pipe segment: pipe_segment_model_anova.R

Target or function Description
get_anova() Runs Monte Carlo ANOVA decomposition for the fitted model
extract_anova_fractions() Extracts named A, S, B, and shared fractions
aggregate_anova_components() Aggregates decomposition outputs across temporal branches
recalculate_anova_components() Converts raw fractions into comparable component summaries

Components:

  • A: variance associated with abiotic predictors.
  • S: variance associated with spatial or spatiotemporal MEM predictors.
  • B: residual association signal from the joint model covariance structure.
  • Shared fractions: variance jointly attributable to combinations of A, S, and B.

Shared fractions are handled explicitly because climate, space, and association structure are not independent in ecological data.

11. Temporal Network Analysis

Temporal analyses also calculate network metrics directly from the binarised community matrix. These metrics provide a model-independent description of co-occurrence structure through time.

NoteTechnical details

Pipe segments: pipe_segment_network_metrics.R and pipe_segment_network_summary_by_age.R

Target or function Description
compute_network_metrics() Calculates network-level metrics from a binary community matrix
plot_network_metrics_by_age() Visualises temporal network trajectories

Metrics:

  • Connectance: proportion of realised links in the network.
  • Nestedness: degree to which specialist taxa occur within subsets of broader assemblages.
  • Modularity: degree to which the network is divided into sub-communities.

12. Prediction and Visualisation

The fitted model stack is also used for prediction and exploratory visualisation. These outputs are useful for checking model behaviour and communicating palaeoecological dynamics, but they remain preliminary while the analysis is being developed.

NoteTechnical details

Relevant helpers include build_prediction_frame(), build_land_prediction_grid(), predict_spatial_resolution_grid_age(), save_prediction_animation(), and save_temporal_trajectory_animation().

Summary and plotting helpers include summarise_spatial_variance_stack(), summarise_spatial_biotic_component(), plot_spatial_variance_stack(), plot_spatial_biotic_component(), and plot_anova_components_by_age().

Current Limitations
