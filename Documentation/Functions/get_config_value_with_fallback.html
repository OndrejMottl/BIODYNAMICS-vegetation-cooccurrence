get_config_value_with_fallbackR Documentation

Get Configuration Value with Fallback

Description

Retrieves a configuration value from the active configuration. If the key is absent there, the function falls back to the same key in a named fallback configuration.

Usage

get_config_value_with_fallback(

  config_section,

  config_key,

  fallback_config = "default",

  file = here::here("config.yml")

)

Arguments

config_section

A single character string naming the configuration section to read, such as '"data_processing"'.

config_key

A single character string naming the configuration setting within 'config_section'.

fallback_config

A single character string naming the configuration to use when the active configuration does not define 'config_key'.

file

Path to the YAML configuration file. Defaults to 'here::here("config.yml")'.

Details

The function is intended for pipelines that keep a shared set of defaults in a dedicated configuration while still allowing project-specific overrides in the active configuration. If 'R_CONFIG_ACTIVE' is unset, the helper reads from '"default"'. Both the active config name and 'fallback_config' must exist in the YAML file.

Value

A scalar configuration value read from the active configuration or from 'fallback_config' when the active configuration does not define 'config_key'.

See Also

[get_active_config()]

Examples

get_config_value_with_fallback(

  config_section = "data_processing",

  config_key = "min_n_samples"

)

get_config_value_with_fallback(

  config_section = "data_processing",

  config_key = "ft_groups_min",

  fallback_config = "project_traits_reference"

)
