Get One Model Tuning Parameter For Scale And Resolution

Description:

     Returns one active model-fitting tuning parameter. Spatial stores
     read per-scale tuning from the model tuning CSV files; non-spatial
     stores fall back to scalar values from `config.yml`.

Usage:

     get_model_tuning_param_for_scale_and_resolution(
       param_id,
       scale_id = get_scale_id_from_store(),
       resolution_id = NULL,
       config_file = here::here("config.yml"),
       dir = here::here("Data/Input/Model_tuning")
     )
     
Arguments:

param_id: A single model tuning parameter name. Must be one of
          `n_iter`, `n_step_size`, `n_sampling`, `n_samples_anova`, or
          `n_early_stopping`.

scale_id: Optional spatial unit identifier. Defaults to
          `get_scale_id_from_store()`. When `NULL`, values are read
          from `config.yml`.

resolution_id: Optional model resolution. Defaults to the active
          `data_processing$taxonomic_resolution` config value. Spatial
          stores pass this to `get_model_tuning_params()`.

config_file: Path to the YAML configuration file. Default:
          `here::here("config.yml")`.

     dir: Directory containing model tuning CSV files. Default:
          `here::here("Data/Input/Model_tuning")`.

Value:

     The requested parameter value. Optional tuning parameters may
     return `NULL`.
